# Prompts para Claude — Hospital Multi-Agent Platform

Este archivo contiene todos los prompts necesarios para generar los 3 agentes simples de un hospital (Appointment, Reminder, Info) y su orquestador, siguiendo la estructura de backend modular y prácticas de orquestación profesional (CrewAI / AutoGen / LiteLLM).

Cada fase incluye la instrucción de generar un documento markdown al finalizar.

---

## Fase 1 — Agente de Citas Médicas (Appointment Agent)

**Prompt:**
```
Using the ARCHITECTURE.md and our multi-agent orchestrator structure,
generate a Python module for the Appointment Agent.

Requirements:
- File: backend/agents/appointment_agent.py
- Functionality: receive patient requests and simulate booking appointments
- Use async/await
- Use Pydantic models for input/output
- Include placeholder integration for CrewAI / AutoGen style agent orchestration
- Add structured logging
- Include a function to return the agent output as a document (markdown) when finished
- Keep modular so it can be reused by the planner/orchestrator
- Simulate available slots with in-memory dictionary
- Return only the Python code for this agent
```

## Fase 6 Frontend

Using the ARCHITECTURE.md, the multi-agent orchestrator, and the 3 hospital agents,
generate a Next.js + ReactJS frontend project for the Hospital AI Platform.

Requirements:
- Project structure:
  - pages/index.tsx  (main dashboard)
  - components/AgentCard.tsx (reusable card for displaying agent output)
  - services/api.ts (functions to call FastAPI endpoints)
- Display the outputs of the 3 agents (Appointment, Reminder, Info) in separate cards
- Fetch data from FastAPI backend asynchronously
- Include loading states and error handling
- Use TypeScript for all files
- Style with Tailwind CSS (or similar lightweight framework)
- Include a function to render markdown generated by agents as visual cards
- Include a feature to refresh workflow results (simulate real-time updates)
- Keep components modular and reusable
- Include a README.md with instructions to run the frontend
- After generating, produce a markdown document summarizing the frontend structure and usage
- Return only the Next.js + ReactJS code and instructions
